CEAOD User Guide on GitHub

This guide helps you contributing your dataset to CEAOD.

  • Step 1: Sign up/sign up to GitHub

    To promote data sharing to help accelerate CEA research, we choose GitHub to be the ideal place for data storage. If the name of GitHub sounds unfamiliar with you at first, no worries, you master it in just a few seconds.

    First, you register your account here: GitHub official site.

    After you set up your account, visit the guide to GitHub as a warmup exercise. It takes only 10 mins: Guide to GitHub.

    When you are reading the guide, pay more attention to terms such as repository, fork or commit. You will need them later ;)

    With that guide, you will learn how to use GitHub by doing. Next time you do the work, the flow is much more natural and intuitive.

    When you are confident, move on to step 2.

  • Step 2: Fork the repository of CEAOD

    Now you fork the repository of CEAOD to your own. Go to the official site of CEAOD on GitHub.

    If you are not sure what it is, come back to the guide in step 1. Reading it carefully and doing sample work will speed your workflow a lot. You fork something when you copy the data from one site to your own for personal using purpose. We keep the data of CEAOD clean for the community. Anyone has to copy data to their sites.

    You can make changes to CEAOD database but more on that later. If you have forked the repository, you do not need to do it again. Forked and ready to make changes? Go to the next step ;)

  • Step 3: Add files to the database

    After you "fork" the repository to your account, you are free to modify it. That's why anyone must fork the repository at first. It introduces the maximum level of flexibility while the data of master files stay safe.

    You can add files to your repository in two ways:

    • Press the "Add files" button to choose files on your computer.
    • Drag and drop files to any screen in GitHub

    We suggest uploading CSV instead of Excel file, as in our data guideline.

    For existing files inside the repository, you can kindly ignore them.

    After you upload your files, you commit it to the database. To commit is to submit changes in an official way to the database.

  • Step 4: Track changes and leave comments

    After you upload your files to the repository and commit it, there will be a new update. You see all added, removed and modified data. All tracked inside a commit. You can see all changes labelled in different colours and symbols. Green is for added, yellow is for modified and red is for removed data.

  • Step 5: Notify the admin

    When you have done with making changes, you can create a pull request to the master repository. To create a pull request is to notify the admin of CEAOD to review changes you made. You can include many commits in one pull request. Or you can commit gradually and notify the admin whenever you like.

    You can also close/reopen your pull requests for convenience.
